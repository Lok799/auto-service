# Автосервис (1С:Предприятие 8.3)

Учебная конфигурация для автосервиса, разработанная на платформе 1С:Предприятие 8.3 в среде EDT. Проект предназначен для демонстрации навыков разработки на 1С и соответствия стандартам.

## Цель проекта
- Показать умение разрабатывать полнофункциональную конфигурацию с несколькими контурами учёта.
- Привести код в соответствие со стандартами 1С (исправлены ошибки, предупреждения, структурированы модули).
- Продемонстрировать навыки работы с EDT, Git, HTTP-сервисами, мобильной платформой.

## Технологический стек
- Платформа 1С:Предприятие 8.3 (управляемое приложение)
- Среда разработки: EDT (Eclipse)
- Система контроля версий: Git (репозиторий на GitHub)
- Мобильная платформа 1С (для мобильного приложения, опционально)

## Функциональные возможности
- **Справочники**: 16 справочников (контрагенты, номенклатура, автомобили, сотрудники, графики работы, договоры, организации, склады, кассы и др.)
- **Документы**: 9 документов (Заказ-наряд, Реализация, Поступление товаров и услуг, Приходный/Расходный кассовые ордера, Начисление зарплаты, Премия, Больничный лист, Учётная политика)
- **Регистры**:
  - Накопления: Товары на складах, Продажи, Учёт затрат, Денежные средства, Заказы клиентов, Материалы заказчиков
  - Сведений: Цены номенклатуры, Кадровая история сотрудников, Учётная политика, Графики работы предприятия
  - Расчёта: Начисления
- **Бухгалтерский учёт**: план счетов, проводки, субконто, регистр Хозрасчетный
- **Зарплатный контур**: регистр расчёта, виды расчёта, графики работы
- **HTTP-сервисы**: endpoints для интеграции с мобильным приложением (регистрация, аутентификация, создание заказ-наряда)
- **Мобильное приложение**: отдельный проект для записи клиентов (см. отдельный README)

## Архитектура
- Модули объектов и менеджеров для всех документов и справочников.
- Общие модули: `БухгалтерскийУчет`, `РаботаСЦенами`, `РаботаСТабличнымиЧастями`, `АутентификацияИАвторизация`, `ОбменСОсновнойБазой` (для мобильной версии).
- Роли: `Автосервис` (основные права), `Администратор` (полные права) – в учебных целях используется одна роль с широкими правами.
- Код структурирован по стандартным областям (`#Область`), устранены предупреждения `module-structure-method-in-regions`, `use-non-recommended-method`, `reading-attribute-from-database` и др.

## Установка и запуск
1. Установите платформу 1С:Предприятие 8.3 (рекомендуется версия 8.3.20 и выше).
2. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/ваш-логин/auto-service.git
3. Откройте проект в EDT (File → Import → 1C:Enterprise → Configuration Project).
4. В свойствах проекта укажите корректную версию платформы.
5. Создайте информационную базу (файловую или серверную) и загрузите конфигурацию.
6. Запустите приложение (Run → 1C:Enterprise Client).

Работа с Git и EDT
Ветка main – стабильная версия.
Ветка Test – для разработки новых функций.

Перед слиянием выполняется Pull Request с проверкой кода.

Особенности и учебные упрощения
Роли "Администратор" и "АдминистраторБезДобавленияПользователей" имеют расширенные права (для упрощения демонстрации).

Часть предупреждений EDT оставлена с комментариями как учебные упрощения.

В мобильном приложении IP-адрес сервера и другие параметры зашиты в коде (в реальном проекте выносятся в константы).
