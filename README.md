# Автосервис (1С:Предприятие 8.3)

Учебная конфигурация для автосервиса, разработанная на платформе 1С:Предприятие 8.3 в среде EDT. Проект предназначен для демонстрации навыков разработки на 1С и соответствия стандартам.

---

## 📌 Цель проекта

- Показать умение разрабатывать полнофункциональную конфигурацию с несколькими контурами учёта.
- Привести код в соответствие со стандартами 1С (исправлены ошибки, предупреждения, структурированы модули).
- Продемонстрировать навыки работы с EDT, Git, HTTP-сервисами, мобильной платформой.

---

## 🛠 Технологический стек

- Платформа 1С:Предприятие 8.3 (управляемое приложение)
- Среда разработки: EDT (Eclipse)
- Система контроля версий: Git (репозиторий на GitHub)
- Мобильная платформа 1С (для мобильного приложения)

---

## 📋 Функциональные возможности

### Справочники (16 шт.)
Контрагенты, Номенклатура, Автомобили, Сотрудники, Графики работы, Договоры, Организации, Склады, Кассы, Должности, Модели автомобилей, Цвета автомобилей, Статьи затрат, Источники информации, Пользователи, Описание цветов для техпаспорта.

### Документы (9 шт.)
Заказ-наряд, Реализация, Поступление товаров и услуг, Приходный кассовый ордер, Расходный кассовый ордер, Начисление зарплаты, Премия, Больничный лист, Учётная политика.

### Регистры
- **Накопления**: Товары на складах, Продажи, Учёт затрат, Денежные средства, Заказы клиентов, Материалы заказчиков.
- **Сведений**: Цены номенклатуры, Кадровая история сотрудников, Учётная политика, Графики работы предприятия.
- **Расчёта**: Начисления.

### Бухгалтерский учёт
План счетов, проводки, субконто, регистр «Хозрасчетный».

### Зарплатный контур
Регистр расчёта, виды расчёта, графики работы.

### HTTP-сервисы
Endpoints для интеграции с мобильным приложением (регистрация, аутентификация, создание заказ-наряда).

### Мобильное приложение
Отдельный проект для записи клиентов (см. README мобильного приложения).

## 🚀 Быстрый старт (тестовая база)

Для быстрого ознакомления с проектом подготовлен файл тестовой информационной базы.

### Скачивание и установка

1. Скачайте файл тестовой базы по ссылке:  
   [Avtoservice_Infobase.dt](https://drive.google.com/drive/folders/1X6PoPPdFGA9f3HkeBSgjI1axQYrS5_Qs?usp=sharing)

2. В Конфигураторе 1С создайте новую пустую информационную базу.

3. Откройте созданную базу в Конфигураторе и выполните:  
   **Администрирование → Загрузить информационную базу...** → выберите скачанный файл `.dt`.

4. Запустите 1С:Предприятие в режиме **«Толстый клиент (управляемое приложение)»**.

5. Для входа используйте пользователя `Администратор` (без пароля).

### Что уже есть в тестовой базе

- Организации, контрагенты, номенклатура, сотрудники.
- Несколько документов (заказ-наряды, реализации, кассовые ордера).
- Настроены графики работы и учётная политика.

## 🔧 Установка из исходников (для разработчиков)

1. Установите платформу 1С:Предприятие 8.3 (рекомендуется версия 8.3.20 и выше).
2. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/Lok799/auto-service.git

3. Откройте проект в EDT (File → Import → 1C:Enterprise → Configuration Project).

4. В свойствах проекта укажите корректную версию платформы.

5. Создайте информационную базу и загрузите конфигурацию.

6. Запустите приложение (Run → 1C:Enterprise Client).   


 Структура репозитория
auto-service/
├── Configuration.xml          # Корневой файл конфигурации
├── Catalogs/                  # Справочники

├── Registers/                 # Регистры

├── HTTP Services/             # HTTP-сервисы
├── Roles/                     # Роли
└── README.md                  # Этот файл
